# Certification Rules

These rules apply to all language paths unless a language-specific
certification document states otherwise.

## Levels
- Certified Beginner: Demonstrates foundational skills and basic testing.
- Certified Intermediate: Demonstrates solid projects, testing, and security.
- Certified Advanced: Demonstrates robust design, profiling evidence, and
  higher rubric scores.

## Evaluation process
1. Reviewer verifies required modules and projects are complete.
2. Reviewer checks evidence artifacts (tests, README, screenshots, logs).
3. Reviewer scores using the rubrics in `assessments/rubrics/`.
4. Reviewer records results in the tracking template.

## Rubric expectations
- Minimum: "Meets" in all criteria for the required rubrics.
- Advanced: "Exceeds" in at least 2 criteria overall, including either
  system design or performance evidence.

## Integrity and originality
- Work must be the learner's own and properly cited.
- AI-assisted work is allowed only if explicitly documented.

## Retake policy
- One resubmission is allowed per attempt.
- Resubmissions must address all reviewer comments.
- If still below standard, a new submission cycle is required.
