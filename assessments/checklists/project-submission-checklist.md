# Project Submission Checklist

Use this for mini projects or mid-course projects.

- [ ] Branch name matches `submission/<learner>/<language>/<project>`.
- [ ] Work is placed under `submissions/<learner>/<language>/<project>/`.
- [ ] `README.md` includes setup, usage, run, and test commands.
- [ ] Features match the prompt and acceptance criteria.
- [ ] Tests or manual verification steps are documented and repeatable.
- [ ] Screenshots or logs are attached when relevant.
- [ ] Code is organized, formatted, and linted.
- [ ] Risks and trade-offs are documented.
- [ ] Self-check against `assessments/rubrics/code-review-rubric.md`.
- [ ] Self-check against `assessments/rubrics/testing-rubric.md`.
