# Capstone Submission Checklist

Use this for final projects and advanced certifications.

- [ ] Branch name matches `submission/<learner>/<language>/capstone`.
- [ ] Work is placed under `submissions/<learner>/<language>/capstone/`.
- [ ] `README.md` includes setup, run, test, and deployment steps.
- [ ] System design notes or diagram are included.
- [ ] Security considerations are documented.
- [ ] Performance notes or profiling output are included.
- [ ] Screenshots or logs show key functionality.
- [ ] Deployment proof is provided when applicable.
- [ ] Self-check against all rubrics in `assessments/rubrics/`.
