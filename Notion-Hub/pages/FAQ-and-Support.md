# ❓ FAQ & Support

## 🆘 Common Questions

### Getting Started

**Q: I've never coded before. Can I really do this?**
**A:** Yes! This course is designed specifically for complete beginners. We start from absolute zero - no prior experience needed. Every concept is explained in plain English. Thousands of people have learned to code starting from scratch. You can too.

**Q: How long does it take to complete?**
**A:** 
- **Per module:** 4-6 hours (part-time)
- **Python track:** 4-6 weeks (part-time)
- **C# track:** 4-6 weeks (part-time)
- **Both tracks:** 8-12 weeks (part-time)

You can go faster or slower - it's self-paced!

**Q: Do I need to be good at math?**
**A:** No! Basic arithmetic (addition, subtraction, multiplication, division) is helpful, but you don't need advanced math. Programming is more about logic and problem-solving than math.

**Q: What tools do I need?**
**A:** 
- A computer (Windows, Mac, or Linux)
- Python (for Python track) or .NET SDK (for C# track)
- A code editor (VS Code or Cursor - both free)
- That's it! Everything else is included in the course.

### Technical Setup

**Q: I'm scared of the terminal/command line. Help!**
**A:** That's totally normal! The terminal is just a way to talk to your computer. We'll teach you exactly what to type, and you'll get comfortable with it quickly. Start with Module 1 - it walks you through everything step-by-step.

**Q: Python/C# won't install. What do I do?**
**A:** 
1. Check Module 1 for detailed installation instructions
2. Make sure you downloaded the right version for your operating system
3. Try restarting your computer after installation
4. Contact support: [SUPPORT_EMAIL] with your operating system and error message

**Q: My code won't run. What's wrong?**
**A:** Common issues:
- **Syntax errors** - Check for missing colons, parentheses, quotes
- **Indentation errors** - Make sure indentation is consistent (Python)
- **File not found** - Make sure you're in the right folder
- **Wrong command** - Check if you're using `python` vs `python3` or `dotnet run`

Read the error message - it usually tells you what's wrong!

**Q: I don't know what to type. Where do I start?**
**A:** 
1. Read the module README first
2. Look at the examples - they show working code
3. Follow the exercises step-by-step
4. Type everything yourself (don't copy-paste at first)
5. If stuck, check the solutions after trying yourself

### Learning & Progress

**Q: I'm stuck on an exercise. What should I do?**
**A:** 
1. Re-read the lesson - you might have missed something
2. Review the examples - see how similar problems are solved
3. Try for 15-20 minutes
4. Check your syntax (colons, parentheses, indentation)
5. Read error messages carefully
6. Take a break and come back
7. Check the solution - but understand why it works

**Q: How do I know if I'm ready for the next module?**
**A:** You're ready when you can:
- Explain the concepts in your own words
- Write code using those concepts without looking everything up
- Complete exercises successfully
- Understand the solutions
- Pass the quiz (or understand what you got wrong)

**Q: Can I skip modules?**
**A:** Not recommended! Concepts build on each other. Skipping modules will make later modules harder. It's better to understand Module 3 fully than rush through Module 7 confused.

**Q: I'm moving too slow/fast. Is that okay?**
**A:** Yes! Everyone learns at their own pace. There's no "right" speed. Focus on understanding, not speed. Consistency matters more than intensity.
