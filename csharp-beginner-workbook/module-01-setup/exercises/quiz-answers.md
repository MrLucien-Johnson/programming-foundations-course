# Module 1 Quiz Answers

## Question 1: What does `Console.WriteLine` do?
**Answer: B** - Prints text to the screen

**Explanation:** `Console.WriteLine` is a command that displays text in the terminal/console window. It's how your program communicates with the user.

---

## Question 2: What command do you use to run a C# program?
**Answer: B** - `dotnet run`

**Explanation:** The `dotnet run` command compiles and runs your C# program. It's the standard way to execute console applications.

---

## Question 3: What is a breakpoint used for?
**Answer: B** - To pause your program so you can inspect what's happening

**Explanation:** A breakpoint is a debugging tool. When your program reaches a breakpoint, it pauses, allowing you to see the current state of variables and step through code line by line.

---

## Question 4: What does compiling mean?
**Answer: B** - Converting your code into something the computer can run

**Explanation:** Compiling is the process of translating your human-readable C# code into machine code that the computer can execute. The `dotnet run` command does this automatically.

---

## Question 5: In C#, text that you want to print must be wrapped in:
**Answer: C** - Quotes `""`

**Explanation:** In C#, text (called a "string") must be wrapped in double quotes. This tells C# that you're providing text, not a command or variable name.

---

## Bonus: True or False Questions

### True or False: You can only use Visual Studio Code for C# programming.
**Answer: False**

**Explanation:** You can use Cursor, Visual Studio Code, or other code editors. Both Cursor and VS Code work great with C#!

### True or False: Every line of C# code must end with a semicolon.
**Answer: False**

**Explanation:** Most C# statements end with a semicolon, but not every line does. Lines with `{ }`, comments, and some directives do not use semicolons. For beginners, remember that most statements (like method calls) need a semicolon.

### True or False: You must create a new project folder for each program you write.
**Answer: False**

**Explanation:** You can create multiple programs in one project, but it is best practice to keep each project in its own folder. This keeps your code organized and avoids conflicts.
