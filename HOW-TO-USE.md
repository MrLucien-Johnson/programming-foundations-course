## 🎯 How to Move Through Modules

### Step-by-Step Process

1. **Read the Module README**
   - Start with `module-XX-name/README.md`
   - Read through the entire lesson
   - Don't skip ahead - concepts build on each other

2. **Study the Examples**
   - Open files in `examples/` folder
   - **For Python:** Run with `python example-01-name.py`
   - **For C#:** Use `dotnet run` in project folder (see Module 1 for setup)
   - Modify them and see what happens
   - Understanding examples helps with exercises

3. **Try the Exercises**
   - Read `exercises/exercise-01.md`
   - Write your own code (don't copy-paste!)
   - Test your code
   - Only check solutions after you've tried

4. **Check Solutions**
   - Compare your solution with `exercise-01-solution.md`
   - Understand why the solution works
   - Learn from differences (there are many correct ways!)

5. **Complete Debug Challenge**
   - Find errors in broken code
   - Practice debugging skills
   - Check solution to see what you missed

6. **Take the Quiz**
   - Test your understanding
   - Check answers
   - Review concepts you got wrong

7. **Move to Next Module**
   - Only move on when you understand the current module
   - It's okay to review previous modules

## 💪 How to Practice Effectively

### Do's ✅

- **Type all code yourself** - Don't copy-paste! Typing helps you learn
- **Run code frequently** - Test as you go, don't wait until the end
- **Experiment** - Change values, try variations, break things

- **Take notes** - Write down concepts you find confusing

- **Don't skip exercises** - Practice is how you build skills
- **Don't just read** - You must write code to learn

## 🔍 When to Check Solutions

### Check Solutions When:

- ✅ You've written your own code and it works
- ✅ You've tried for 15-20 minutes and are stuck
- ✅ You want to see alternative approaches
- ✅ You've completed all exercises and want to compare

### Don't Check Solutions When:

- ❌ You haven't tried writing code yet
- ❌ You've only been stuck for 2 minutes
- ❌ You want to "just see how it's done"
- ❌ You're feeling lazy (push through!)

**Remember:** Looking at solutions too early prevents learning. Struggle is part of the process!

## 🚧 How to Avoid Getting Stuck

### If You're Stuck:

1. **Re-read the lesson** - You might have missed something
2. **Review examples** - See how similar problems are solved
3. **Check your syntax** - Missing colons, parentheses, or indentation?
4. **Read error messages** - Python/C# tells you what's wrong
5. **Take a break** - Sometimes stepping away helps
6. **Try a simpler version** - Break the problem into smaller parts
7. **Check solutions** - After genuine effort, solutions are learning tools

### Common Stuck Points:

- **Syntax errors** - Check colons, parentheses, quotes, indentation
- **Logic errors** - Code runs but does wrong thing (trace through step-by-step)
- **Type errors** - Trying to do math with strings? Convert types!
- **Index errors** - Lists start at 0, not 1
- **Infinite loops** - Forgot to update counter in while loop?

**Remember:** Getting stuck is normal. Every programmer gets stuck. The skill is learning to get unstuck!
