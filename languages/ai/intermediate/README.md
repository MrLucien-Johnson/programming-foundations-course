# AI — Intermediate Curriculum

## What you’ll build
Practical AI features and systems that are **repeatable, testable, and safe**—without relying on a
specific vendor or tool.

## Focus
You’ll build reliable LLM features: structured outputs, RAG foundations, eval/testing discipline,
guardrails, agentic workflows, and operational cost/latency planning.

## Principles (this curriculum’s “golden rules”)
- **Start with a spec**: goal, inputs, outputs, constraints, and failure modes.
- **Test early and often**: create a small evaluation set before you optimize prompts.
- **Iterate methodically**: change one variable at a time; keep notes.
- **Safety by default**: handle sensitive data, refusal behavior, and hallucinations.
- **Measure**: track quality, cost, and latency; define budgets.

## Recommended workflow
- Complete modules in order.
- Each module ends with a mini-project.
- Then complete `projects/` (two mini-projects + a capstone).
- Treat submissions like an engineering PR: docs, tests/evals, and clear run steps.
