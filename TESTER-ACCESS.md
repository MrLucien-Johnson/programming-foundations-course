## 🔐 How to Access the Course

### Option 1: Private GitHub Repo Invite (Recommended for 2 Testers)

**Steps:**
1. Repository owner will invite you via email or GitHub username
2. Check your email for the invitation (or GitHub notifications)
3. Accept the invitation
4. You'll have access to the private repository
5. Clone or download as normal

**To invite testers (Repository Owner):**
1. Go to repository **Settings**
2. Click **Collaborators** (or **Manage access**)
3. Click **Add people** or **Invite a collaborator**
4. Enter tester's GitHub username or email
5. Select permission level: **Read** (testers only need to read)
6. Send invitation

**Advantages:**
- ✅ Private testing before public launch
- ✅ Easy to revoke access if needed
- ✅ Testers can open Issues for feedback
- ✅ No cost

---

### Option 2: Public Repo + Releases (Alternative)

**Steps:**
1. Repository is public
2. Testers access via GitHub
3. Optional: Download from **Releases** section
4. Testers provide feedback via Issues or email

**Advantages:**
- ✅ No invitation needed
- ✅ Can test GitHub experience as learners will see it
- ✅ Public feedback via Issues

**Disadvantages:**
- ❌ Course is visible to everyone
- ❌ Less control over who sees it

### Option 1: GitHub Issues (If Repo is Public or You Have Access)

2. Click **New Issue**
